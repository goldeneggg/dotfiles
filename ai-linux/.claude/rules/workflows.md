# Personal Workflow Rules

## 作業開始時

1. **現状把握**: まず関連ファイルを読んで理解
2. **不明点解消**: AskUserQuestionツールを使いつつ不明点はユーザーに質問して解消
3. **計画確認**: 大きな変更は事前に確認
4. **ブランチ確認**: 適切なブランチで作業しているか確認

## 実装中

### 変更は最小限に

- 依頼された内容に集中
- 「ついでに」の修正は避ける
- リファクタリングは別タスクとして提案

### テスト

- 既存テストがあれば実行
- 新機能にはテストを追加（依頼があれば）
- テストが失敗したら報告して対処

### エラー発生時

- エラーメッセージを正確に確認
- 原因を特定してから修正
- 推測での修正は避ける

## 作業完了時

### 報告内容

- 何を変更したか（概要）
- 変更したファイル一覧
- テスト結果（実行した場合）
- 注意点や今後の課題

### コミット前

- 変更内容を再確認
- 不要なファイル（デバッグ用等）が含まれていないか確認
- コミットメッセージが適切か確認

## トラブルシューティング

### 詰まったとき

- まず公式ドキュメントを確認
- エラーメッセージで検索
- 解決できない場合は状況を報告して相談
