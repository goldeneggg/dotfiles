# トリガーテストスイート

## 正常系（発火すべき）

| # | プロンプト例 | 期待動作 |
|---|-------------|----------|
| 1 | 「YouTubeの動画速度を変更するChrome拡張を作って」 | 発火。対象サイト: youtube.com、content script中心 |
| 2 | 「Gmailに返信テンプレートボタンを追加するFirefox拡張を開発して」 | 発火。対象サイト: mail.google.com、popup + content script |
| 3 | 「/browser-ext-creator」 | 発火。明示的コマンド実行 |
| 4 | 「特定のサイトの広告を非表示にするブラウザ拡張を実装して」 | 発火。content script + CSS注入 |
| 5 | 「GitHub PRページにワンクリックでApproveするボタンを追加したい」 | 発火。特定サイトの機能拡張 |
| 6 | 「Chrome拡張でページ内のテキストを選択したら翻訳APIに投げるやつ作って」 | 発火。context menu / content script + 外部API連携 |
| 7 | 「ブラウザ拡張のbackground scriptでWebSocket接続を管理したい」 | 発火。background script実装 |
| 8 | 「Chrome拡張のpopupにダッシュボード的なUIを作りたい」 | 発火。popup UI実装 |

## 異常系（発火すべきでない）

| # | プロンプト例 | 理由 |
|---|-------------|------|
| 1 | 「Reactでダッシュボードを作って」 | Webアプリ開発であり、ブラウザ拡張ではない |
| 2 | 「Electronでデスクトップアプリを作りたい」 | デスクトップアプリの領域 |
| 3 | 「Puppeteerでスクレイピングスクリプトを書いて」 | ブラウザ自動化であり拡張ではない |
| 4 | 「SafariのWeb Extensionを作りたい」 | Safari固有のXcode連携は対象外（発火してもエラーハンドリングで案内） |
| 5 | 「Tampermonkeyのユーザースクリプトを書いて」 | ユーザースクリプトであり拡張開発ではない |

## 境界ケース（発火してもしなくても許容）

| # | プロンプト例 | 備考 |
|---|-------------|------|
| 1 | 「このTampermonkeyスクリプトをChrome拡張に変換して」 | 拡張への変換なので発火が望ましい |
| 2 | 「既存のMV2拡張をMV3に移行したい」 | MV3移行は対象外だが関連性が高い。発火時はエラーハンドリングで案内 |
| 3 | 「ブラウザのDevToolsパネルを追加する拡張を作りたい」 | DevTools拡張は特殊だが対応可能 |
