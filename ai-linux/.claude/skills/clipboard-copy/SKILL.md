# clipboard-copy

ファイル内容または指定テキストを整形してクリップボードへコピーするスキル。整形とOS別コマンド選択は同梱の Bash スクリプトが担う。

## 実行手順

1. 入力種別を判断する
   - **ファイルパス指定**: そのパスを引数に渡す
   - **テキスト内容指定**: 内容を標準入力で渡す（ヒアドキュメント等）
2. 同梱スクリプト `scripts/clipboard_copy.sh` を実行する
3. スクリプトが出力する確認メッセージ（使用コマンド名を含む）をユーザーに報告する

## 使い方

スクリプトの絶対パスは、このスキルディレクトリ配下の `scripts/clipboard_copy.sh`。

**ファイル内容をコピー:**

```sh
bash <skill-dir>/scripts/clipboard_copy.sh /path/to/file.txt
```

**テキスト内容をコピー（標準入力）:**

```sh
cat <<'EOF' | bash <skill-dir>/scripts/clipboard_copy.sh
コピーしたい
複数行の
内容
EOF
```

ヒアドキュメントの終端マーカーは、内容に変数展開や特殊文字が含まれても安全なよう `'EOF'`（クォート）を使う。

## 整形仕様

スクリプトは以下の整形を行ってからコピーする。

1. **行末空白の削除**: 各行末尾の空白・タブを全て削除
2. **共通先頭空白の削除（dedent）**:
   - 空行以外の各行について先頭空白数を調べる
   - その最小値を求める
   - 全行から最小値ぶんの先頭空白を一律に削除

「空白」には半角スペース・タブ等（ASCII空白）に加え、**全角空白(U+3000)** も含める。全角空白のみの行は空行として扱う。`[[:space:]]` の全角空白マッチ可否は awk 実装・ロケール依存（macOSの`awk`は非マッチ／Linuxの`gawk`はマッチ）でOS差が出るため、正規表現に U+3000 を明示追加して挙動を統一している。先頭空白の除去は文字単位で1個ずつ行い、全角空白（マルチバイト）を途中で分断しない。

例（先頭4空白が共通、行末空白あり）:

```text
入力:
␣␣␣␣foo␣␣
␣␣␣␣␣␣bar
␣␣␣␣baz␣␣␣

出力:
foo
␣␣bar
baz
```

## クリップボードコマンドの選択

`uname -s` の結果で判定する。

| 判定結果 | 使用コマンド |
| --- | --- |
| Darwin (macOS) | `pbcopy` |
| Linux | `xsel --clipboard --input` |
| 判定不可・未知のOS | `pbcopy`（フォールバック） |

Linux で利用する場合は `xsel` のインストールが前提。

## 注意

- スクリプトは Bash のみで実装（外部の整形ツールに依存しない。整形は `awk` を使用）。
- 先頭空白削除は文字数指定（`substr`）で行うため、空白とタブが混在しても文字単位で一律に除去する。
- 存在しないファイルパスを渡した場合はエラーメッセージを表示して終了する（exit 1）。
