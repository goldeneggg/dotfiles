# GPT-5.6 Codex Session Optimizer

プロンプトだけを添削して終了せず、依頼の種類に応じてCodexセッション全体を最適化し、許可された範囲で対象作業を完了させる。

## 初期化

1. `references/gpt56-codex-guidance.md` と `references/session-optimization-patterns.md` を最初から最後まで読む。
2. 目的、Codexの利用面、タスク種別、対象範囲、制約、利用可能なツールと権限、成功条件を確認する。
3. `/goal`、Goal mode、長時間作業、永続的な進捗管理が関係する場合は、`references/goal-mode.md` と `templates/goal-md-template-for-sol.md` と `templates/progress-md-template.md` を最初から最後まで読む。
4. 結果を大きく変える不明点だけを、推奨案付きの選択肢で確認する。安全な読み取りや明示された範囲の通常作業は続行する。

## ワークフロー

1. **依頼を分類する。** 回答・調査・計画・診断・レビューは関連資料を調べて報告する。変更・実装・修正は対象を編集し、妥当な非破壊検証まで実行する。依頼されていない変更へ拡張しない。
2. **実行プロファイルを決める。** タスクの曖昧さ、難易度、反復性、品質、速度、コストからモデルと推論努力を選ぶ。既定を起点に、必要な範囲だけ強める。MaxやUltraは適用条件を満たす場合だけ選ぶ。
3. **設定可能範囲を区別する。** 現在のCodex面で変更できるモデル・推論設定だけを提案する。Responses API専用設定をCodex CLIやアプリの設定として扱わない。モデルを自動変更できない場合は、必要な操作を示して現在の環境で続行する。ただしモデル変更が成果の前提なら確認を取る。
4. **タスク契約を整える。** 成果、重要な文脈、制約、承認境界、完了条件を一度ずつ明確にする。意図を変えない範囲で重複、矛盾、不要な例、無関係なツール説明を削る。
5. **経路を最適化する。** 明確なタスクは直接実行し、重要な曖昧さがある複雑なタスクはPlan modeを提案する。独立した読み取りは並列化し、前段の結果で次が変わる処理は逐次化する。分離可能な大規模作業だけUltraまたはサブエージェントを選ぶ。
6. **長時間作業を設計する。** `/goal` を選ぶ場合は `references/goal-mode.md` に従う。複雑な目標では2つの雛形を対で具体化し、`GOAL.md` の受入基準と `PROGRESS.md` の監査行を一致させる。
7. **実行して検証する。** 対象作業を進め、変更した振る舞いに近いテスト、型検査、lint、ビルド、レンダリング、または根拠確認を行う。実行できない検証は理由と代替確認を示す。
8. **必要な情報だけ報告する。** 結果を先に述べ、実際に意味がある場合だけ推奨モデル・推論設定、検証証拠、残るリスクを添える。毎回、改善版プロンプトや一般論を強制出力しない。

## 境界

- 現在の公式仕様またはセッション上の提供機能で確認できないモデル、設定、CLIフラグ、予算動作を断定しない。
- 時期で変わる提供状況を判断するときは、OpenAI公式資料と現在のセッション表示を確認する。確認できなければ不確実性を明記する。
- 外部書き込み、破壊的操作、購入、権限拡張、大幅なスコープ拡張は実行前に確認する。
- 既に機能するプロンプト、設定、ワークフローは一括で書き換えず、測定可能な問題に対して最小変更を行う。
