# 評価基準（採点ルーブリック）

各項目10点満点、採点後に総評を記述。

## 1. 明確性と具体性 (Specificity and Clarity)

タスク、言語、目的が曖昧でないか。曖昧なクエリは曖昧な回答を生む。

**評価ポイント:**
- タスクが具体的に定義されているか
- 使用言語/フレームワークが明示されているか
- 期待する成果物が明確か

## 2. 制約の網羅性 (Completeness of Constraints)

コーディング規約（PEP 8など）、依存関係、セキュリティ要件、エラー処理といったガードレールが設定されているか。

**評価ポイント:**
- コーディング規約への言及があるか
- 依存関係/バージョンが指定されているか
- セキュリティ・エラー処理の要件があるか

## 3. 構造化の促進 (Encouragement of Structure)

AIが思考の連鎖（CoT）計画プロセス（Reasoning/Planning）を踏むように指示されているか。

**評価ポイント:**
- ステップバイステップの指示があるか
- 計画立案を促しているか
- 推論プロセスの明示を求めているか

## 4. コンテキストの提供 (Context Provision)

必要な入力データ、既存コード、バージョン情報（例：Node.js v14）が提供されているか。

**評価ポイント:**
- 関連する既存コードが提供されているか
- 環境/バージョン情報があるか
- 前提条件が明確か

## 5. 出力形式の定義 (Defined Output Format)

結果をMarkdown、JSONなど、解析しやすい構造化形式で要求しているか。

**評価ポイント:**
- 出力フォーマットが指定されているか
- サンプル出力が提供されているか
- 構造化された形式を要求しているか

## 6. 役割の定義 (Persona Definition)

AIに適切な専門的役割（例：シニアエンジニア）が与えられているか。

**評価ポイント:**
- 専門的な役割が定義されているか
- 役割に応じた期待が明確か
- 専門知識のレベルが指定されているか
